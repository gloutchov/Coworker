Introduction

Coworker is an experimental project designed to provide a local AI companion for people working on a computer. It uses local Open Weight LLMs tuned for standard office laptops and supports daily tasks.

Main features:

  • Brainstorming (with or without reasoning).
  • Information retrieval based on:
    • the model knowledge base,
    • local document archives,
    • previous conversations.
  • Transcription of audio files or live conversations.
  • Creation of org charts, flowcharts, and mind maps from chats or local documents.
  • Use of MCP tools (user managed).

The system is built to protect privacy by keeping data local and avoiding third party servers.

Getting started

Starting Coworker

To start the program:

  1. Double click coworker.bat.
  2. A terminal window opens to load the local server.
  3. The interface opens in your browser (initially standby).

On macOS you can start the app by double clicking coworker.command.

At startup the server detects hardware and sets performance parameters (context window and max tokens). It also initializes the database for RAG and local preferences.

Note: on slower systems you may need to reload the page to see saved chats.

Warning: you can minimize the terminal window but do not close it.

Interface

The Coworker interface is divided into three areas.

Mode area (left)

Lets you switch modes, view chat history, and open configuration. Available modes:

  • Chat AI (open mode): free questions based on model knowledge and loaded documents. Supports:
    • reasoning models (Thinking),
    • graphics (org charts, flowcharts, mind maps),
    • configured MCP servers,
    • Wikipedia search (only network access),
    • live transcription from the browser,
    • a mini editor for notes or short documents.
  • Chat Documents (closed mode): answers only from indexed documents in a dedicated folder. Graphics and mini editor are available here too.
  • Chat History (closed mode): answers based on saved and indexed conversations.

Note: the app starts in Chat AI mode.

Question area (center)

Includes the input field and options that change by mode.

Answer area (right)

Shows:

  • the active model,
  • status/progress bar,
  • response area,
  • buttons to print, save, delete, or clear the chat.

Note: the system keeps context until you clear the area, enabling a chain of thought conversation.

How it works

Coworker offers three usage modes, described below.

Mode: Chat AI

This is the default mode.

Getting started:

  • Enter a question in Cosa vuoi sapere?.
  • For a first run, try: "Cosa puoi fare per me?" and press Send.

The model responds and remembers the conversation so you can ask follow up questions.

Available actions:

  • Save chat: saves the conversation to history.
  • Print chat: generates a print document (header and footer) and allows print or PDF.
  • Clear chat: clears the response area without saving.
  • Stop: interrupts the current response.

History management:

  • saved chats appear in history,
  • you can reopen them by clicking,
  • you can rename them with double click,
  • to delete: open the chat and click Delete chat.

Wikipedia search

In Chat AI you can enable Wikipedia search to query only Wikipedia. The model searches Wikipedia first and, if nothing is found, uses its own knowledge base. Source links are listed when Wikipedia is used.

MCP (Model Context Protocol)

MCP lets AI models access external resources and go beyond basic chat limits, connecting to external systems (ERP, email, calendar, CAD, etc.).

Coworker includes an MCP client and communicates with MCP servers via HTTP and STDIO protocols.

When enabled, the interface shows connected MCP servers. To query them, type @ followed by the server name.

Warning: MCP servers must be installed and configured by the user with proper credentials and permissions.

Mode: Chat Documents

In this mode the model answers only based on indexed documents.

Main steps:

  1. Copy documents into the dedicated folder (configurable).
  2. Run indexing.
  3. Ask questions about the indexed documents.

Characteristics:

  • Closed mode: no answers outside indexed content.
  • Supports Graphics and the mini editor.
  • Supports OCR for images (when a multimodal model is available).

Mode: Chat History

In this mode the model answers based only on saved conversations. Chats are indexed automatically.

Configuration

Configuration lets you adapt Coworker to your needs. Main sections:

User editable options (main):

  • active models and custom LLM models,
  • documents folder and models folder,
  • response delivery (streaming) and thinking visibility,
  • context and max tokens (fixed/dynamic),
  • Excel/CSV read limits,
  • preferred graphics format (SVG/Mermaid/PlantUML),
  • email for Wikipedia search,
  • additional MCP services,
  • external APIs (provider, base URL, model, API key).

LLM models

Select the default model, add or remove models, and manage available local models.

Default parameters

Initial settings to ensure smooth performance and hardware compatibility.

LLM parameters

Typical settings: context window, max tokens, and performance/quality balance.

Note: higher max tokens improve response capacity but use more resources. Coworker uses conservative values, but they can be customized.

Document management

Select the document folder, index or reindex content, and manage uploaded documents.

Excel/CSV limits

Excel/CSV files can be large. To avoid incorrect indexing, configurable limits are available for max rows and max columns. Default: 300 rows, 20 columns.

Note: empty values or 0 reset to server defaults.

Wikipedia search

Enable or disable Wikipedia access. This is the only network access allowed by the system. To enable it you must add your email in the dedicated field, used as contact for Wikipedia requests.

System prompt

Displays the internal system prompt (not user editable) that guides model behavior.

Graphics

Enable graphic generation from text prompts. Available formats:

  • SVG: static vector output for logos, badges, and simple schemes. Good for immediate export.
  • Mermaid: text markup for flowcharts, org charts, and mind maps, easy to render.
  • PlantUML: UML oriented markup, useful for UML diagrams and alternatives to Mermaid flowcharts.

Audio management

Controls file transcription, live transcription from the browser, and recording settings and limits.

Note: audio features require ffmpeg available on PATH.

Text to speech

Converts text documents to audio.

MCP services

Configure connected MCP servers and invocation modes. Use the Add MCP button to paste a JSON service configuration.

Main JSON fields (recommended minimum):

  • name: service identifier (used with @name).
  • label: display label.
  • description: service description.
  • type: service type (e.g. mcp_http, mcp_stdio).
  • instructions: usage instructions.
  • endpoint/url: HTTP endpoint (for HTTP services).
  • command and args: startup command (for STDIO services).
  • optional: headers, payload, env, timeout, prompt_mode, prompt_arg.

MCP demo in /tools:

  • A demo MCP server is available in lai/tools/micro mcp server/.
  • STDIO start (default): python3 micro_mcp_server.py
  • HTTP start: python3 micro_mcp_server.py --transport http --host 127.0.0.1 --port 8000
  • HTTP endpoint: http://127.0.0.1:8000/mcp

Note: install fastmcp if missing with python3 -m pip install fastmcp.

External APIs

Configure compatible external services when enabled. Typical fields: provider, base URL, model name, API key.

User paths

Manage local paths for documents, models, cache, and user data.

User information

General user information and system status.

User preferences

Set response options (streaming or full response), show or hide thinking blocks, choose fixed or dynamic context, choose fixed or dynamic max tokens, and set display preferences.

Details:

  • Response delivery: streaming shows tokens progressively; full response appears only when complete.
  • Fixed/dynamic context: fixed keeps the default window to reduce memory/latency; dynamic expands context for long inputs or histories.
  • Fixed/dynamic tokens: fixed uses the server default; dynamic adjusts the limit between min and max based on available context.

Language switch

The interface is bilingual. Click the language icon in the top right to switch Italian/English.

This manual

The manual is always available from the app via the Help link in the bottom right.

Appendix

LLM models used

Coworker uses local LLMs chosen for a balance of performance and reliability on office hardware.

Current models:

  • Qwen 2.5 3B Instruct (4 bit): main text model (writing, document analysis, translations, summaries). Context up to 32k tokens.
  • Gemma 3 4B Instruct (4 bit): multimodal model for text + images/OCR. Context up to 128k tokens.
  • Phi 3.5 4B Instruct (4 bit): specialized in logic, math, and structured reasoning. Context up to 128k tokens.
  • Qwen 3 4B Instruct (4 bit): dedicated to deep thinking. Context up to 32k tokens, higher resource usage.

All models are Open Weight and run locally. Sizes (3-4B parameters) are tuned for office hardware.

Core concepts:

  • Prompt: user request.
  • Context window: how much information the model keeps in memory (question, answers, documents).
  • Token: minimal processing unit (not always a full word).

Balancing prompt, tokens, and context improves responses and efficiency. Coworker manages these parameters automatically based on hardware.

How to write effective prompts

  • Clear goal: avoid generic requests. Prefer targeted tasks (summary, data extraction, comparison).
  • Context: specify role, goal, and scenario (presentation, internal procedure, technical document).
  • Documents and images: state whether answers must rely only on uploaded documents or can include general considerations.
  • Thinking: with reasoning enabled, ask precise and focused questions. Break complex problems into steps.
  • Output format: specify lists, tables, structured output, or narrative text.
  • Iteration: refine the first response with follow up questions.
  • Critical review: Coworker is a support tool and does not replace user responsibility.

Installation

Before starting the program, run the setup scripts.

Windows:

  1. Double click setup_windows.bat.
  2. Wait for dependency installation to finish.

macOS (recommended order):

  1. Double click setup_mac.command.
  2. Then double click install_llama_mac.command.
  3. Wait for both scripts to complete.

What the setup scripts do (summary):

  • Prepare the Python environment (virtualenv) and install required dependencies.
  • Download and configure local components needed to run the models.

Coqui model download (recommended before first run):

  • Run the warm-up script: python3 tools/warmup_coqui_tts.py --accept-tos
  • This script pre-downloads the Coqui TTS model to avoid delays on first use.

Note: audio features require ffmpeg available on PATH (not included in this repository).

Recommended installation:

  • macOS (Homebrew): brew install ffmpeg
  • Windows (winget): winget install Gyan.FFmpeg
  • Windows (Chocolatey): choco install ffmpeg

If macOS does not open .command files, run from Terminal:

  1. Open Terminal in the project folder.
  2. Make the scripts executable:
chmod +x setup_mac.command
chmod +x install_llama_mac.command
chmod +x coworker.command
  1. Run the setup scripts:
./setup_mac.command
./install_llama_mac.command
  1. Start the app:
./coworker.command

Glossary of technical terms

  • AI: system that understands text, analyzes information, and produces responses.
  • Chain of Thought: internal reasoning producing deeper answers, but uses more resources.
  • Chat AI: open mode based on general knowledge and loaded content.
  • Chat Documents: closed mode based only on indexed documents.
  • Chat History: closed mode based on saved conversations.
  • Context Window: temporary model memory (question, answers, recent content).
  • GPU / CPU: hardware components that perform the computations.
  • Hardware: physical components affecting performance.
  • Indexing: process to prepare documents, images, or audio for querying.
  • Knowledge Base: information available to the model at a given time.
  • LLM: large language model.
  • Local: runs entirely on the user's computer.
  • Man in the Middle: the AI supports, but the user decides.
  • MCP: protocol to connect AI to external tools.
  • Specialized model: optimized for specific tasks.
  • Multimodal: works with text, images, and audio.
  • OCR: text recognition from images or scanned documents.
  • Open Weight: model parameters are available for local use.
  • Performance: speed and efficiency based on model and hardware.
  • Prompt: user input request to the model.
  • System prompt: internal instructions guiding the model.
  • Quantization: technique to reduce model memory usage.
  • RAG: retrieval from local documents plus generation.
  • SVG: vector format for simple graphics.
  • Thinking: deep reasoning mode.
  • Token: minimal unit of processing.
  • Token Max: maximum number of tokens in output.
  • Live transcription: audio recording converted to text.
  • Wikipedia API: controlled access to Wikipedia.
